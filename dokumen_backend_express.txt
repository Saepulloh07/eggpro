======================================================
DAFTAR ISI
1. Arsitektur & Teknologi
2. Struktur Direktori

======================================================

1. ARSITEKTUR & TEKNOLOGI
-------------------------
- Runtime: Node.js
- Framework: Express.js (v4)
- Bahasa: TypeScript
- ORM: Prisma (untuk type-safety dan migrasi database yang mudah)
- Database: PostgreSQL (Relational DBMS, sangat cocok untuk sistem ERP/Akuntansi ini)
- Autentikasi: JSON Web Token (JWT) + bcryptjs untuk hashing password
- Validasi Input: Zod (terintegrasi dengan Express via middleware)
- Logging: Winston / Morgan

2. STRUKTUR DIREKTORI

3. KONFIGURASI DATABASE (Gambaran Prisma Schema)
------------------------------------------------
Contoh abstraksi model Prisma berdasarkan struktur_data.txt:

model User {
  id             String   @id @default(uuid())
  name           String
  username       String   @unique
  email          String   @unique
  password       String
  role           Role     @default(WORKER)
  assignedHouses String[] // Bisa juga relasi Many-to-Many ke House
  salary         Float?
  createdAt      DateTime @default(now())
}

enum Role {
  SUPER_ADMIN
  ADMIN
  WORKER
}

model House {
  id            String   @id @default(uuid())
  name          String
  capacity      Int
  area          Float?
  managerId     String?
  // Relasi
  productionLogs ProductionLog[]
  flocks        FlockBatch[]
}

model ProductionLog {
  id                  String   @id @default(uuid())
  houseId             String
  date                DateTime
  eggCount            Int
  feedConsumed        Float
  feedInventoryItemId String
  mortality           Int
  mortalityCause      String?
  totalButir          Int
  house               House    @relation(fields: [houseId], references: [id])
}

// Dan seterusnya untuk Inventory, Transaction, Account, Journal, dll.
// (Semua relasi akan menggunakan foreign keys yang diatur di Prisma)

4. MODUL & ROUTING (ENDPOINTS)
------------------------------

A. Modul Auth (/api/auth)
   - POST /api/auth/login    -> Authenticate user & return JWT
   - GET  /api/auth/me       -> Get current logged-in user profile

B. Modul Users (/api/users) - Akses: SUPER_ADMIN
   - GET    /api/users       -> List all users
   - POST   /api/users       -> Create new user
   - PUT    /api/users/:id   -> Update user (termasuk reset password)
   - DELETE /api/users/:id   -> Delete user

C. Modul Master Data (House, Flock) (/api/master)
   - GET/POST /api/master/houses
   - GET/POST /api/master/flocks

D. Modul Produksi (/api/production) - Akses: ALL (terbatas houseId untuk WORKER)
   - GET    /api/production        -> Ambil data produksi (filter by date/house)
   - POST   /api/production        -> Input produksi harian
     (Logic di Controller: Saat post produksi, backend harus OTOMATIS:
      1. Kurangi stok pakan di inventory
      2. Tambah stok telur di inventory
      3. Catat jurnal akuntansi (Beban Pakan pada Persediaan Pakan)
      4. Buat record mortalitas jika ada kematian)

E. Modul Inventory & Gudang (/api/inventory)
   - GET  /api/inventory           -> Lihat stok
   - POST /api/inventory/mutation  -> Input mutasi stok (Beli, Pakai, Pindah)

F. Modul Keuangan & Akuntansi (/api/finance)
   - GET  /api/finance/accounts    -> Lihat Chart of Accounts
   - GET  /api/finance/journals    -> Lihat Jurnal Umum
   - GET  /api/finance/trial-balance -> Neraca Saldo / Laba Rugi
   - POST /api/finance/transactions -> Catat transaksi kas (Otomatis buat jurnal)
   - GET/POST /api/finance/apar    -> Manajemen Hutang Piutang

G. Modul Penjualan (/api/sales)
   - POST /api/sales               -> Catat penjualan
     (Logic: Kurangi stok telur, catat transaksi masuk/Kas, catat Jurnal Penjualan)

5. MIDDLEWARE
-------------

A. authMiddleware:
   Mengecek header `Authorization: Bearer <token>`.
   Jika valid, decode token dan inject `req.user = decoded`.
   Jika tidak, throw 401 Unauthorized.

B. roleMiddleware(roles: string[]):
   Mengecek `req.user.role`.
   Jika role user tidak ada di array `roles`, throw 403 Forbidden.
   Contoh: router.post('/users', authMiddleware, roleMiddleware(['SUPER_ADMIN']), UserController.create);

C. errorHandler:
   Menangkap error global (terutama error Prisma/Zod) dan mengembalikan respons JSON standar.
   { "status": "error", "message": "...", "details": [...] }

6. STRATEGI IMPLEMENTASI
------------------------

FASE 1: Setup & Autentikasi
- Inisialisasi Express & Prisma
- Buat seed database untuk INITIAL_ADMIN dan DEFAULT_ACCOUNTS
- Implementasi sistem Login dan JWT

FASE 2: Master Data & Inventory
- CRUD User, House, Flock
- CRUD Inventory items (Barang, Pakan, Telur)

FASE 3: Core Transaction (Produksi & Penjualan)
- Pembuatan endpoint POST /api/production (Sangat krusial: harus dibungkus dengan Prisma Transaction `$transaction` agar jika gagal di tengah (misal potong stok sukses tapi jurnal gagal), maka seluruh operasi di-rollback).
- Pembuatan endpoint POST /api/sales

FASE 4: Modul Keuangan Lengkap
- Endpoint Jurnal Umum, Laporan Keuangan (Trial Balance, Laba Rugi)
- Manajemen AP/AR (Hutang Piutang)
- Penyusutan Aset
