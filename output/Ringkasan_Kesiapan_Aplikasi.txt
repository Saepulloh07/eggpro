2. EMPAT CRITICAL BLOCKER TERBESAR (WAJIB DIPERBAIKI SEBELUM DEPLOY)
================================================================================
1. Cacat Sinkronisasi Penghapusan Offline (Offline Deletion Leak)
   - Transaksi yang dihapus pengguna saat offline tidak pernah terhapus di database 
     pusat MySQL karena backend sync hanya mengandalkan kueri UPSERT. Ini memicu data 
     historis ganda dan bias biaya fiktif.
2. Inkonsistensi Kode Bagan Akun Akuntansi (COA Code Mismatch)
   - Selisih format kode akun 4-digit (frontend) vs format hierarkis standar 
     (backend database seed) merusak integritas penyusunan Buku Besar dan Trial Balance.
3. Ketiadaan Validasi Kunci Periode di Sisi Server (Server-Side Period Lock Bypass)
   - Penutupan buku bulanan dapat dilewati secara mudah oleh request API langsung karena 
     validasi penguncian tanggal transaksi hanya dilakukan di sisi web browser (client-side).
4. Distorsi Perhitungan HPP Telur Menggunakan Last Price
   - HPP telur dihitung menggunakan harga pembelian terakhir, bukan Moving Average. 
     Fluktuasi harga pakan instan akan langsung mengacaukan perhitungan laba bersih 
     historis secara retroaktif.
