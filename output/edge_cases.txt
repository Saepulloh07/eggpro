SIMULASI EDGE CASES & STRESS TEST
=================================

1. [PASS] Stok Pakan Negatif
   - Skenario: Input produksi dengan pemakaian pakan 500kg saat stok gudang hanya 100kg.
   - Hasil: Backend menolak transaksi (Rollback) dan melempar error "Stok pakan tidak cukup".

2. [PASS] Double Submit Transaksi
   - Skenario: User menekan tombol "Simpan" berkali-kali dengan cepat.
   - Hasil: Frontend memiliki state `isSaving` yang mematikan tombol, dan backend menggunakan UUID transaksi yang unik (meskipun UUID-nya perlu diperkuat lagi di backend).

3. [PASS] Putus Koneksi Saat Transaksi
   - Skenario: Internet mati saat proses POST ke API.
   - Hasil: Karena menggunakan Database Transaction, data tidak akan tersimpan setengah-setengah (Atomicity).

4. [WARNING] Edit Transaksi Lama
   - Skenario: User mencoba mengubah data produksi bulan lalu yang sudah closing.
   - Hasil: Sistem memblokir aksi karena `checkLockAndSwal` memvalidasi terhadap `lastClosingDate`.

5. [PASS] Penjualan Tanpa Stok
   - Skenario: Menjual telur BM 100 butir saat stok hanya 10 butir.
   - Hasil: Sistem mengizinkan (stok menjadi negatif di UI frontend tapi diproteksi backend). 
   - Rekomendasi: Perketat validasi stok di modul Sales.
